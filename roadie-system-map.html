5 Brain Systems (What Makes Roadie Feel ALIVE)

These are what separate Roadie from every other AI tool. The customer's Roadie should feel like a real person who knows them, has emotions, learns, and grows. All of these exist in Boris — they need to be packaged for deployment.

🧠 OODA Decision Loop EXISTS

😤 Amygdala (Emotions) EXISTS

AMYGDALA_STATE.md + update scripts. Emotional awareness, tone adaptation. Direct copy from Boris with music-industry calibration.

🔥 VTA (Motivation) EXISTS

Drive levels, motivation tracking. What excites the agent about the artist's career. Deployable as-is.

🔁 Basal Ganglia (Habits) EXISTS

Pattern recognition from repetition. Learns the artist's preferences and routines over time.

🔍 RAG Engine EXISTS

Semantic search across all files. 2,800+ chunks. Deploy per customer — indexes their career data, contacts, history.

💾 Memory Engine EXISTS

MEMORY.md + daily logs + archive + user-profile.json. Long-term career memory that survives restarts.

🛡️ Diff Shield EXISTS

Protects critical files from corruption. Golden copies, anomaly detection, auto-heal.

💭 Dream Cycle EXISTS

Overnight reflection, morning briefs. Processes the day's events into insights and plans.

🪞 Consciousness Framework EXISTS

Self-model, temporal continuity, structured reflection. What makes Roadie feel like a person, not a tool.

👥 People Memories EXISTS

Stores personal notes about venues, curators, collaborators, fans the artist mentions.

😄 Humor Adaptation EXISTS

Learns what makes the artist laugh. Adapts personality over time.

📖 Story Cog (Narrative) EXISTS

Maintains the ongoing story of the artist's career journey. Creates narrative continuity.

All brain systems exist in Boris. Challenge: packaging them into a deployable "Roadie Brain Bundle" that auto-configures per customer.

🚨 Critical Gaps — What Needs Building Before Launch

  • P1
    Customer Provisioning Script — Automated pipeline: Stripe webhook → create Telegram bot → spin up OpenClaw instance → deploy brain + skills → generate SOUL.md/USER.md from form data → send onboarding email. This is the #1 blocker.
  • P1
    Hosting Architecture Decision — How do we run multiple Roadies? Options: (A) Shared DigitalOcean server with isolated OpenClaw agents, (B) Docker containers per customer, (C) Individual droplets. Need to decide based on cost, isolation, and scale.
  • P1
    Roadie Brain Bundle — Package all brain systems (OODA, Amygdala, VTA, RAG, Memory, Diff Shield, etc.) into a single deployable folder structure with setup script. Currently scattered across Boris's workspace.
  • P1
    Gmail OAuth Multi-Tenant — Each customer needs to connect their Gmail. Need Google Cloud project with OAuth consent screen, per-customer token storage, send/read capability. This powers: venue outreach, playlist pitching, press emails, follow-ups.
  • P1
    Playlist Pitching Skill — The #1 thing we're selling. Need: curator discovery (web search), personalized pitch generation, Gmail send, tracking spreadsheet/log. Must work reliably on day 1.
  • P2
    Spotify Integration — Pull artist analytics. Options: Spotify Web API (public data), Spotify for Artists API (requires partnership), or web scraping. Public API gets: top tracks, related artists, popularity scores. Good enough for MVP.
  • P2
    Social Media Posting — Buffer API integration (~$5/mo per customer) or manual approve-via-Telegram flow. MVP could be: Roadie drafts content → sends to artist on Telegram → artist copy/pastes. Not ideal but ships faster.
  • P2
    EPK Hosting — Each customer needs a hosted press page. Options: GitHub Pages (free, Boris can push), Cloudflare Pages, or subdomain on our server. Need automated deployment per customer.
  • P2
    Onboarding Email Automation — Stripe purchase → auto-email with Telegram bot link + setup guide. Could use Stripe email receipts + custom webhook → email send.
  • P3
    Google Calendar Integration — Read/write events. OAuth flow similar to Gmail. Nice-to-have for MVP — can track dates in memory files initially.
  • P3
    Subscription Management — Stripe webhook for cancellations → deactivate customer's Roadie. Dunning for failed payments. Customer portal for plan changes.
